Best practices

  1. если проект большой и в 1 репе N компонентов -- то в одной ветке нельзя работать над 2+ компонентами
  2. git clone --bare + много чекаутов для разных веток —-- лучше чем git stash
  3. нельзя использовать git add . или -[Aa] :)
  4. .gitignore для платформы
  5. в фич-ветке история коммитов может отличаться до мержа
  6. git submodules — говно
  7. gitflow — не говно, рассказать про стратегии бранчевания
